development/testing environments best practices

In short:

  • accounts and instances associated with them should only be run when they need to be - be it developer accounts that only need to run during office hours or test account that only need to run during testing. They should never sit idle.

When provisioning non-production environments, such as dev or test, it is crucial to ensure that they are active only when in use. Keeping these resources running 24/7 is wasteful, as running instances consume energy even while idle (see utilization).

Each type of environment should run on a different schedule. Development environments should only run during working hours of the development team, test environments should only run when tests are being performed, and so on.

Most cloud providers offer automated lifecycle management tools for environments. In cases where environments need to run on inconsistent schedules, on-demand instances can be utilized.
