in short

  • serverless functions allow you to run processes without a need to provision any resources.
  • when it comes to performing purely computational tasks (tasks that require close to no data to be loaded) they are significantly more energy efficient than provisioned servers
  • serverless computing functions are inherently stateless, so they should not be used to perform processes that require loading large files from memory

serverless functions allow you to run processes without a need to provision any resources. This allows you to avoid under provisioning resources, because all of the computing is being done in a single large scalable server that is being maintained by cloud providers.

Lambda functions are not always the best option thou. They are inherently stateless, meaning that if a function needs to load a large file from memory it’s going to have to do it every time it’s run. That is significantly less efficient than performing this calculation on a individually provisioned server, where this data can be kept alive in memory
