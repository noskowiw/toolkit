ARM processors are based on a different architecture than x86 processors and use a different instruction set.

ARM processors were designed with energy efficiency in mind

for certain processes there chips can offer a significant energy efficiency advantage

Two ways to run programs on arm chips - native arm builds and x86 builds running with a instruction set transcriber

There’s a significant performance disadvantage when running x86 builds with a transcriber, and this method should be avoided at all costs

comparison/analogy - new MacBook's run on arm chips - m1s and m2s. old proprietary software runs slower because it uses a transcriber: in this case Rosetta

arm chips are nothing new native arm builds are widely available for commonly used libraries and applications, thou you might have to specify that you want to use them

programs running on arm chips can also benefit from other optimizations made during the build process.
