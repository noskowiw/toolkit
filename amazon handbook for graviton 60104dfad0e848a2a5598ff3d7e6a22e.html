amazon handbook for graviton

ARM processors are based on a different architecture than x86 processors and use a different instruction set. ARM processors were designed with low power consumption in mind and for certain processes these chips can offer a significant energy efficiency advantage.

It’s important to know there are two types of programs that can be run on arm chips - native arm builds and x86 builds running with a instruction set transcriber. There’s a significant performance disadvantage when running transcribed x86 builds and this method should be avoided at all costs

<>

comparison/analogy - new MacBook's run on arm chips - m1s and m2s. Old proprietary software runs slower because it uses a transcriber: in this case Rosetta

ARM chips are nothing new and native arm builds are widely available for commonly used libraries and applications, thou you might have to specify that you want to use them. Programs running on arm chips can also benefit from other optimizations made during the build process, but these changes should be considered on case by case bases
