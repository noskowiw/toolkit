Code profilers

in short:

  • Even an idling virtual machine is responsible for some amount of energy consumption, leaving resources unutilized should be avoided at all cost.
  • Underutilizing resources does not allow them to reach their full theoretical energy efficiency. When choosing what hardware to use when deploying your application, always consider the amount of computing power it requires and choose the appropriate instance type accordingly.

Every CPU clock cycle, memory allocation or disk read comes with its own energy expenditure. Although running any large-scale application will require trillions of these operations to executed, making sure that your application doesn't perform them unnecessarily is an important part of making your code more sustainable.

Profiling code is the best way of gaining insight into your application's performance and potential avenues of improvement. These tools measure metrics associated with the execution of your program such as time of execution, allocated memory, CPU utilization or number of function calls to the function and display them in a legible way.

Being aware of these metrics allows you to identify bottlenecks and recourse hungry elements of your code, allowing you to direct your attention where it’s the most needed. Some good ways to increase the efficiency of those fragments of code include:

  • Reduce memory consumption and execution time of frequently used functions
  • Improve memory management in general
  • Apply algorithms with a proper time and memory complexity in places where they are most impactful
  • Cache frequently-used functions
