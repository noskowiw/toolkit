Programming languages

In short:

  • Energy efficiency of your program is highly dependent on the programming language you’re using.
  • Low-level compiled languages are usually more efficient, but the extra effort required to maintain the code written in them may outweigh the benefits of using them.
  • Codebases don’t have to be written in only one language, and large products can probably benefit from being divided into different parts written in appropriate languages

Energy efficiency of a program is highly dependent on the programming language it’s written in . Additional layers of abstraction present in high-level languages require the system to perform more computations, in turn decreasing the energy efficiency of programs which were written in them.

This however does not mean that using a low-level programing language is always a good idea. Aside from simply being less practical, writing software close to the hardware requires the more developer time and, especially in the case of very rudimentary languages like C, can introduce a number of bugs and problems that will need to be addressed throughout the products lifecycle. This in turn negates the benefits of the increase in performance. Moreover, if a program is not written correctly it may end up being less efficient than the one written in a more high-level language.

Fragments of code that don’t see much traffic or don’t perform many complex calculation will likely not see much of an improvement from being written in a low-level language. However making sure that more complex, highly utilized fragments of code are written in as fast of a language as they practically can be can have a great impact on the overall energy efficiency of the product they’re a part of. Most cloud codebases are not monolithic, which makes it possible to implement different parts of those solutions in different languages. This way you can ensure that the product you’re running is as efficiently as it possibly can be while not requiring extra maintenance.
