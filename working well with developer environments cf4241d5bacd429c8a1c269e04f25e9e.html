When provisioning non-production environments (e.g. dev, test), it is important to ensure that they are only active when they can be realistically utilized. Keeping these resources provisioned around the clock is very wasteful, as running instances still use up energy even while idling (see utilization). Each type of environment should run on a different schedule. Development environments should only be running during working hours of the development team, test environments should only be running when tests are being performed, and so on.

Practically every cloud provider allows users to utilize some kind of automated lifecycle manager for their environments. In the case of environments that need to be run with inconsistent schedules, on-demand instances can be utilized.
