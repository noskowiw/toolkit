working well with developer environments

date:

author: Wojciech Noskowiak

When provisioning non-production environments, such as dev or test, it is crucial to ensure that they are active only when in use. Keeping these resources running 24/7 is wasteful, as running instances consume energy even while idle (see utilization). Each type of environment should run on a different schedule. Development environments should only run during working hours of the development team, test environments should only run when tests are being performed, and so on.

Most cloud providers offer automated lifecycle management tools for environments. In cases where environments need to run on inconsistent schedules, on-demand instances can be utilized.
