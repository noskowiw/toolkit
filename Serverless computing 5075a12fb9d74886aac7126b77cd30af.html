Serverless computing

In short:

  • Serverless computing functions allow you to run processes without the need to provision any resources.
  • These functions are significantly more energy efficient than provisioned servers when it comes to performing purely computational tasks (tasks that require close to no data to be loaded).
  • They are inherently stateless, so they should not be used to perform processes that require loading large files from memory.

Serverless functions allow you to run processes without the need to provision any resources. The calculations required to execute these functions are performed on clusters maintained by cloud providers. This improves the energy efficiency of your solutions for two reasons :

  • The virtualization overhead is distributed between every serverless process currently running in the datacenter.
  • Cloud providers have a strong incentive to make those clusters run as energy efficiently as possible, so there is a high likelihood that the processing is being performed as efficiently as it can be.

However, Lambda functions are not always the best option. They are inherently stateless, meaning that if a function needs to load a large file from memory it is going to have to do it every time it is run. That is significantly less efficient than performing this calculation on a individually-provisioned server, where this data can be kept accessible in system memory.
